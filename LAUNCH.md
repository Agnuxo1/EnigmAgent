## dev.to

**Title:** I Built an Encrypted Vault for AI API Keys That Never Touches the Cloud

**Tags:** #opensource #security #ai #javascript #privacy

**Text:**

Every new AI tool asks for API keys. We paste them into `.env` files, browser extensions, CLI configs. It's messy and insecure.

I built **EnigmAgent** — a local encrypted vault that keeps your AI credentials secure, synced, and completely offline.

## What makes it different

| Feature | EnigmAgent | Typical password manager |
|---------|-----------|------------------------|
| Encryption | AES-256-GCM (browser) | Server-side |
| Sync | P2P mesh (Gun.js) | Centralized cloud |
| Backend | None | Yes |
| Quantum-safe | CRYSTALS-Kyber/Dilithium ready | No |

## How it works

1. Open the vault in your browser
2. Add API keys — encrypted with a master password
3. Your agents read from the vault via the EnigmAgent API
4. Keys sync across devices via Gun.js P2P mesh

## Try it

**Live demo:** https://agnuxo1.github.io/EnigmAgent/
**GitHub:** https://github.com/Agnuxo1/EnigmAgent

## The tech stack

- Vanilla JavaScript (no build step)
- Gun.js for P2P graph database
- WebCrypto API for encryption
- MIT licensed
