EnigmAgent · end-to-end verification

Domain binding matters. This page must be served from the same domain as the secret's domain field. Loading the file over file:// will not work because the extension's content script is excluded from file:// URLs. Run a local server: 
python -m http.server 8080
then visit http://localhost:8080/tests/smoke-test.html, and bind your test secrets to localhost.

Steps

  1. Load the EnigmAgent extension in your browser (see README).
  2. Open the vault and unlock it (or create a new one).
  3. In the vault chat, add two test secrets:
    add DEMO_TOKEN @localhost demo-value-12345
    add DEMO_USERNAME @localhost alice
  4. Come back to this page and click Submit below.
  5. The badge (bottom-right) should flash "✓ submitted with real values", and the JSON panel below the button should show the decrypted values.
  6. Open DevTools → Console and verify no real value was ever logged before submit.

These fields contain secret references. EnigmAgent resolves them from the encrypted vault at submit time — the values are never exposed before that moment.

Negative-path verification

Add a secret bound to a different domain and confirm the extension refuses to resolve it on this page:

add ONLY_ON_GITHUB @github.com should-never-appear-here

Paste {{ONLY_ON_GITHUB}} into the token field and submit. The badge should show ⚠ domain_mismatch — the value is never revealed.
