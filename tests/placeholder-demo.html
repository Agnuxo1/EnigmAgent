EnigmAgent · end-to-end smoke test

Domain binding matters. This page must be served from the same domain as the secret's domain field. Loading the file over file:// will not work because the extension's content script is excluded from file:// URLs. Run a tiny local server: 
python -m http.server 8080
then visit http://localhost:8080/tests/placeholder-demo.html, and bind your test secrets to localhost.

Steps

  1. Load the EnigmAgent extension in your browser (see README).
  2. Open the vault, create it with a username and a ≥ 8-char password.
  3. In the vault chat, type:
    add DEMO_TOKEN @localhost demo-value-12345
    add DEMO_USERNAME @localhost alice
  4. Come back to this page and click Submit below.
  5. The badge (bottom-right) should flash “✓ submitted with real values”, and the JSON panel should show the decrypted values — which this page would have POSTed.
  6. Open DevTools → Console. Before submit it will never have logged the real value. Look for any rogue output.

These fields intentionally contain placeholders. EnigmAgent must substitute them at submit time.

Expected negative-path tests

Bind a secret to a different domain to verify that the extension refuses to resolve it here:

add ONLY_ON_GITHUB @github.com should-never-appear-here

Then paste {{ONLY_ON_GITHUB}} into the token field and submit. You should see a ⚠ domain_mismatch badge, not the value.
